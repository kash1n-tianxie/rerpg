# 🤝 共同研究・開発への貢献ガイドライン (CONTRIBUTING)

本研究プロジェクトに関心をお寄せいただきありがとうございます！**rerpg** は、極限のJRPGリソース制約下において、人間、強化学習（RL）、および大規模言語モデル（LLMs）の意思決定動作、脆弱性パターン、協調戦略を比較検証するための実験プラットフォームです。

研究者やAI開発者の皆様からの、以下の方向性での貢献を心よりお待ちしております：
1. **新しい意思決定エージェントの検証**（例：他のオープンソースLLMの統合、SACやRainbow DQN等の異なる強化学習手法のテスト）。
2. **強化学習環境におけるエキスパート報酬設計の拡張**。
3. **人間プレイヤーによる対局ログや実験データの共有**。

---

## 🛠️ 実験における一貫性と整合性（レッドライン）

実験データの公平性と再現性を学術的に保証するため、提出されるすべてのコードやデータは以下のルールを厳守する必要があります：

1. **ルールの共通化（一貫性）**：
   * すべての実験対象（RL、LLM、人間）は、一律で **5000 HP** のボス設定を使用する必要があります。
   * 各対局は一律で最大 **50ターン** に制限されます。
   * `rpg_boss_fight_env.py` および `services/BattleEngine.ts` に定義されているダメージ計算、消費AP、および回復量などのコアロジックを無断で変更することは禁止されています。

2. **データ収集の基準**：
   * 強化学習の訓練は一律で **100,000ステップ** 実行し、TensorBoardログを出力する必要があります。
   * 最終的な評価（Evaluation）は、決定論的ポリシー（Deterministic Policy）下で少なくとも **100エピソード** 実行し、平均DPEおよびDESを報告する必要があります。

## 🚀 貢献手順

### 1. バグ・脆弱性（Exploits）の報告
JRPG戦闘システムにおいて、ルール設計の不整合や予期せぬ脆弱性（例：特定の行動パターンで無敵状態になる等）を発見された場合は、Issueを起票してください。その際、以下の情報を記載してください：
* **問題の概要**：どのような脆弱性または整合性の欠如が発生しているか？
* **再現手順**：コマンドラインでのコマンド、またはWeb上での操作手順。
* **学術的影響**：理論上の最大累積ダメージ限界（5037.0点）の計算に影響を与えるか？

### 2. 新しいエージェントの追加
優れた DES (意思決定効率スコア) を達成するエージェントを開発された場合は、ぜひ Pull Request (PR) を送信してください：
1. **本レポジトリをFork**し、フィーチャーブランチを作成します：
   ```bash
   git checkout -b feature/your-agent-name
   ```
2. **エージェントの実装**：
   * Pythonの強化学習コードの場合は、ルートディレクトリに訓練スクリプトを格納し、`--eval` オプションによる確定評価に対応させてください。
   * Web上のLLMエージェントの場合は、`constants.ts` にプロンプト戦略を定義し、`App.tsx` の選択肢に追加してください。
3. **DES指標の報告**：
   * PRの概要に、100エピソード実行時の平均DPEとDESの平均値・標準偏差を明記してください。
4. **PRを送信**し、レビューとルールの対照チェックを行います。
