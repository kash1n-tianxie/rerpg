# 🤝 参与本研究项目的开源贡献指南 (CONTRIBUTING)

感谢您对本学术项目的关注！**rerpg** 是一项旨在研究人类、强化学习（RL）和大语言模型（LLM）在极端受限及即死高压环境下决策表现对比的学术项目。

我们非常欢迎并鼓励其他学术研究人员或游戏 AI 爱好者加入，共同探讨和扩展以下方向：
1. **测试新的智能体决策策略**（例如：对接新的开源 LLM、尝试不同的强化学习算法如 SAC, Rainbow DQN 等）。
2. **贡献或扩展专家启发式奖励系统**。
3. **提交人类测试数据和统计日志**。

---

## 🛠️ 实验规范与要求

为了保证学术研究结果的公平性和可对比性，所有贡献的代码或数据需要遵循以下一致性红线：

1. **游戏规则一致性**：
   * 所有决策组别（RL、LLM、人类网页游玩）必须统一使用 **5000 HP** 的 Boss 设定。
   * 单局游戏回合上限统一为 **50 回合**。
   * 不得随意篡改 `rpg_boss_fight_env.py` 和 `services/BattleEngine.ts` 中的伤害、AP 恢复或冷却规则，任何影响一致性的修改将不予通过。

2. **数据收集的一致性**：
   * 强化学习对比训练需统一跑满 **100,000 步**（默认设定），并导出 TensorBoard 统计文件。
   * 最终对比的评估（Evaluation）需在确定性策略（Deterministic Policy）下运行至少 **100 次完整的对局（Episode）**。

## 🚀 贡献步骤

### 1. 报告 Bug 与建议
如果您在运行 RL 环境或网页端时发现任何防御判定逻辑漏洞（Exploit）或者游戏规则不一致的问题，请提交 Issue。格式如下：
* **问题描述**：发生了什么错误或逻辑漏洞？
* **重现步骤**：提供完整的终端运行指令或操作步骤。
* **学术影响**：该问题是否会影响之前的理论伤害上限（5037点）的计算？

### 2. 提交新的决策算法 (Agent)
如果您开发了新的一代 (Generation) 的 Agent，并取得优异的决策效率成绩，欢迎提交 Pull Request (PR)：
1. **Fork 本仓库** 并创建您的特性分支 (Feature Branch)：
   ```bash
   git checkout -b feature/your-new-agent
   ```
2. **编写代码**：
   * 如果是 Python 端强化学习算法，请将训练脚本存放在根目录下，并确保它能通过 `--eval` 参数加载并以确定性策略测试 100 轮。
   * 如果是网页端的 LLM，请在 `constants.ts` 增设您的 Prompt 战略常数，并在 `App.tsx` 的选择下拉菜单中添加该选项。
3. **导出评估指标**：
   * 请在 PR 描述中贴出您的 100 轮 DPE (平均单局伤害) 和 DES (决策效率得分) 的均值与标准差。
4. **提交代码** 并发起 PR，我们的团队会对您的实验数据和代码一致性进行审核。
