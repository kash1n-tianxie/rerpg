# 🌐 フロントエンド起動・デプロイガイド (DEPLOYMENT_GUIDE)

本项目は React + TypeScript + Vite で開発された、美麗なJRPGドット絵フロントエンドWebアプリケーションを含んでいます。人間プレイヤーによるブラウザでの手動プレイのほか、Gemini APIと連携させることで、LLMエージェントによる自動プレイが可能です。

---

## 1. ローカルでの起動 (開発サーバー)

フロントエンドは Node.js 環境下で動作します。あらかじめ Node.js (推奨 v18+) がインストールされていることを確認してください。

### 1.1 依存パッケージのインストール
プロジェクトのルートディレクトリで、以下のパッケージインストールコマンドを実行します：
```bash
npm install
```

### 1.2 ローカル環境変数の設定 (.env)
Google Gemini API を連携させるために、ローカル環境変数を設定します。
1. プロジェクトのルートディレクトリに `.env.local` という名前のファイルを作成します：
   ```bash
   touch .env.local
   ```
2. ファイルに以下の通りAPIキーを記述します（Viteの仕様上、接頭辞 `VITE_` が必要です）：
   ```env
   VITE_GEMINI_API_KEY=あなたのGoogle_Gemini_API_Key
   ```

> [!TIP]
> Gemini APIキーは [Google AI Studio](https://aistudio.google.com/) で無料で取得できます。APIキーを設定しない場合でもWeb画面は動作しますが、AI自動プレイおよび戦敗分析機能はローカルのモック戦略（Fallback）に切り替わります。

### 1.3 ローカル開発サーバーの起動
以下のコマンドで開発サーバーを起動します：
```bash
npm run dev
```
起動に成功すると、ローカルポート `3000` でサーバーが立ち上がります。
ブラウザで [http://localhost:3000](http://localhost:3000) を開きます。

* 注：`vite.config.ts` で `host: '0.0.0.0'` と設定されているため、同じLAN環境内の他の端末や仮想マシンからもテストプレイ画面にアクセス可能です。

## 2. Web画面の3つの実験モード

Webアリーナに入ると、ヘッダーで以下の操作モードを切り替えることができます：

### 👤 手動プレイモード (MANUAL)
* **対象**: 人間プレイヤー。
* **操作**: 各ターン、Arthur（タンク）、Merlin（魔法使い）、Ellie（回復士）の順にスキルをクリックし、対象（味方またはボスのポートレート）を選択して `EXECUTE TURN` をクリックします。
* **目的**: 即死処刑による時間的・心理的プレッシャーとAP制約下における、人間の操作ミスや注意の逸れに関するデータを収集します。

### 🤖 AI自動プレイモード (AI AUTO - Gen 0 & Gen 5)
戦闘の意思決定をLLMに委ねます。Gemini APIを呼び出す2つのエージェント：
* **初級AI (Gen 0)**：`constants.ts` 内の `GEN0_STRATEGY` を使用します。ゲームのルールやボスの行動サイクルを一切教えないシンプルなプロンプトであり、算術的な幻覚を起こしやすい特徴があります。
* **熟練AI (Gen 5)**：`constants.ts` 内の `GEN5_STRATEGY` を使用します。詳細なルール、ボスの行動サイクル、チーム連携の指示を含んでおり、高度な推論による意思決定を行います。

### 🧠 戦術反思と自己進化モード (EVOLUTION)
* **概要**: 大規模言語モデルによる戦敗分析とプロンプトの動的更新。
* **メカニクス**: 熟練AIでプレイし敗北した際、戦術分析ターミナルが起動します。直近20行の戦闘ログを `gemini-2.5-pro` に送信し、敗北原因を特定します。
* **進化**: 分析結果から動的な戦術制約ルール（例：`ARTHUR MUST TAUNT ON TURN 4`）を抽出し、次回の対戦時にシステムプロンプトに動的にマージします。これにより「世代的な戦術進化」を擬似的に実現します。

## 3. ビルドと GitHub Pages へのデプロイ

遠隔で人間プレイヤーによるテストを実施したり、Webアプリを公開するために、GitHub Pages にデプロイする手順は以下の通りです：

### 3.1 プロダクションビルド
```bash
npm run build
```
ビルドに成功すると、`dist/` ディレクトリに静的ファイルが出力されます。

### 3.2 GitHub Pages へのデプロイ
`gh-pages` ツールを使用してデプロイを行います：
1. 開発依存パッケージとしてインストール：
   ```bash
   npm install gh-pages --save-dev
   ```
2. `package.json` にデプロイスクリプトを追加：
   ```json
   "scripts": {
     "predeploy": "npm run build",
     "deploy": "gh-pages -d dist"
   }
   ```
3. デプロイを実行：
   ```bash
   npm run deploy
   ```
4. サブディレクトリのパス問題が発生する場合は、`vite.config.ts` で `base: '/あなたのレポジトリ名/'` を適切に設定してください。
