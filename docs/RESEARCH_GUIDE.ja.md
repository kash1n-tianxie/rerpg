# 🔬 卒業論文科学的実験ガイド (RESEARCH_GUIDE)

本研究は、極めて過酷で容認度の低いRPG戦闘環境において、**大規模言語モデル (LLM)**、**強化学習 (RL)**、および**人間プレイヤー (Human)** の意思決定パフォーマンスを比較し、高ストレスおよびリソース制限下における異なる知能システムの意思決定特性、脆弱性パターン、および協調的戦術を探索することを目的としています。

本ガイドは、今後の実験設計、データ収集、および論文執筆のための統一的な学術フレームワークを提供します。

---

## 1. コアとなる研究課題と仮説

本研究は、以下の3つの主要な研究仮説に基づいています：

1. **仮説一：LLMにおける「高圧下の幻覚と記憶の減退」**  
   低ストレス環境では完璧な論理推論を示しますが、厳格な状態追跡（ボスの行動サイクルの把握）や精密な計算（残りAPの監視）を要求される高ストレス環境下では、戦闘ログ（コンテキスト）の増加に伴い、**計算の幻覚**や**コンテキスト喪失**が発生し、意思決定の精度が急速に低下します。

2. **仮説二：強化学習における「報酬ハッキング」と「過学習による劣化」**  
   詳細なエキスパート報酬（Detailedモード）の下では、エージェントは設計された報酬式の抜け穴を利用する傾向があります（例：敗北ペナルティを回避するために、早期に自爆して対局を終了する）。逆に、シンプルな報酬（Baselineモード）の下では、エージェントは極めて最適なヘルスしきい値で自爆するといった、より高次元の創発的協調戦略を発見しやすくなります。

3. **仮説三：人間の意思決定における「心理的ストレス」による注意の逸れ**  
   人間プレイヤーは直感的な適応能力に優れますが、即死攻撃のプレッシャー下では計算ミス（例：残り1 APの数え間違い）を誘発しやすくなります。また、恐怖による防衛本能から過剰に回復へリソースを割いてしまい、最適な攻撃チャンスを逃す傾向があります。

## 2. 実験設計と評価指標 (Metrics)

論文において説得力のある学術的図表を提供するため、各実験グループで少なくとも **100回独立した対局（Episode）** を実行し、以下の指標を記録します：

### 2.1 定量評価指標一覧

| 分類 | 学術的指標 | 指標の説明 | 計算式 | 学術的な意義 |
| :--- | :--- | :--- | :--- | :--- |
| **効率性** | **Damage Per Episode (DPE)** | エピソードあたりの累積ダメージ | $\sum \text{Damage}$ | ボスに対する最終的な攻撃進捗を測定します。 |
| **生存力** | **Survival Turns (ST)** | 生存ターン数 | 終了時のターン数 | 即死サイクルに対するタンクと回復の防衛連携能力を評価します。 |
| **品質** | **Damage Per Turn (DPT)** | ターンあたりの平均ダメージ | $\text{DPE} / \text{ST}$ | 防御と攻撃の間における資源分配の効率性を表します。 |
| **協調** | **Healer Sacrifice Rate (HSR)** | 回復士の犠牲率 | Ellieが死亡しArthurが生存したエピソード割合 | 利他的な自己犠牲協調戦略の創発を観察します。 |
| **戦術** | **Self-Destruct Timing (SDT)** | 自爆時のボス残りHP割合 | 自爆時のボス残りHP % | エージェントが自爆ダメージを最大化する最適なタイミングを把握したかを測定します。 |
| **調和** | **Healing Efficiency Ratio (HER)** | 回復/ダメージ調和指数 | $\text{Total Healing} / \text{Total Damage}$ | 生存と攻撃のテンポのバランスを測定します。 |

### 2.2 オラクル基準（Oracle Baseline）と意思決定効率（DES）

不確実性下における意思決定効率を科学的に評価するため、グローバルDPソルジャーによって証明された**理論上最大累積ダメージの上限値 5037.0 点**を **Oracle Baseline** として導入します。

論文執筆の際、各実験グループの**意思決定効率スコア (Decision Efficiency Score, DES)** を算出してください：

$$\text{DES} = \frac{\text{Mean DPE}}{\text{DP Oracle Max Damage (5037.0)}} \times 100\%$$

> [!NOTE]
> **学術的文脈**：Cycle 0 および Cycle 3 は攻撃対象がランダムに選ばれるため、プレイヤーは生存を確実にするために防衛行動をとるトレードオフを迫られます。実際の Mean DPE と DP Oracle (5037.0) のギャップは、**「不確実性下において生存を保障するために犠牲にした“戦略的機会費用（Opportunity Cost of Uncertainty）”」**を定量的に表しています。

## 3. 対照実験の実行手順

### ステップ 1: 古いデータの削除
訓練ログや古いモデルキャッシュを削除し、クリーンな環境を用意します：
```bash
rm -rf models/* tensorboard_logs/* test_tensorboard/*
```

### ステップ 2: 強化学習の対照訓練の実行
1. **Baseline モデルの訓練**（100,000ステップ）：
   ```bash
   python3 train_baseline.py --steps 100000 --name PPO_Baseline
   ```
2. **Detailed モデルの訓練**（100,000ステップ）：
   ```bash
   python3 train_rl.py --steps 100000 --name PPO_Detailed
   ```
3. TensorBoard ([http://localhost:6006](http://localhost:6006)) を開き、`custom/total_damage_mean` と `custom/survival_turns_mean` の収束状況を監視します。

### ステップ 3: 評価指標の収集
訓練されたモデルをそれぞれ確定論的ポリシー下で100エピソード実行し、指標を記録します：
```bash
# Baseline の評価
python3 train_rl.py --eval --model models/baseline_ppo

# Detailed の評価
python3 train_rl.py --eval --model models/rpg_boss_ppo
```

### ステップ 4: LLMおよび人間のデータの収集
1. **LLMエージェント**: Webフロントエンドから、Gen 0 および Gen 5 プロンプトでそれぞれ50回自動プレイを実行し、指標を集計します。
2. **人間プレイヤーのテスト**: ゲーム経験のあるプレイヤーを5〜10人招待し、Web上でそれぞれ10回プレイしてもらいます。Web画面上の「STATS」パネルからデータをエクスポートします。

## 4. 論文執筆と図表の推奨構成

論文内において、以下の図表の構成を計画することを推奨します：

1. **学習収束曲線 (Baseline vs. Detailed)**  
   * **横軸**: Training Steps (0–100k)  
   * **縦軸**: Episodic Reward / Mean Damage  
   * Baselineが収束は遅いものの、最終的にDetailedの最大パフォーマンスを上回る現象を分析します。

2. **意思決定崩壊レーダーチャート (Human vs. LLM vs. RL)**  
   * 数値的ストレス耐性、記憶維持能力、操作一貫性、APルール順守率、戦術的Taunt成功率の5つの次元で能力ギャップを視覚化します。

3. **自爆創発プロット (SDT 散布図)**  
   * 自爆トリガー時のターン数（横軸）とダメージ（縦軸）をプロットし、PPO Baselineが明示的な指示なしに後期ターンでの自爆ピークをいかに自己発見したかを考察します。
